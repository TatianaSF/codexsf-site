# Content Guide

This repository treats Markdown as the public content database for Codex SF.

All content must be safe to publish before it is committed.

## Public Content Folders

- `content/hackkit/` contains HackKit playbook pages.
- `content/resources/` contains reusable public templates and resources.
- `content/events/` contains public event notes, recaps, highlights, and demo project summaries.

## Markdown Format

Use a short frontmatter block at the top of each Markdown file:

```markdown
---
title: Example Title
description: Short public description.
order: 1
status: Public
---

### Section heading

Public-safe content goes here.
```

Keep descriptions short and useful. Use `order` to control display order inside a collection.

## HackKit Pages

HackKit is public and open-source. Add pages that help organizers and builders reuse practical event patterns.

Good HackKit content includes:

- public event formats
- public timelines
- public checklists
- public judging guidance
- public pitch guidance
- public templates
- public lessons that do not identify private people or partners

## Resources

Resources should be reusable and public. They can provide structure, prompts, and examples, but they must not include real private contacts, private assignments, budgets, or internal plans.

## Event Content

Event pages should focus on public information:

- public announcements
- public recaps
- public highlights
- demo summaries approved for publication
- public links that are ready to share

Do not publish attendee lists, private feedback, private logistics, internal planning notes, or partner-only details.

## Writing Tone

Use a clear, practical, community-friendly, builder-focused tone.

Prefer concise language that helps people understand what Codex SF is, how HackKit works, and how to participate in public community programming.

## Never Add

- secrets or credentials
- private contacts
- budgets
- speaker notes
- volunteer assignments
- internal strategy
- private checklists
- partner-specific internal planning
- unpublished sensitive drafts
- private forms or private admin links
